# 京豆查询工具

## 功能

- 自动读取当前 CRM 组的关闭量数据，支持查询今天关闭量或昨天加今天关闭量。
- 按整组或指定追踪人筛选待查询数据。
- 支持通过 `jpos` 或 `kfuad` 两种请求来源查询京豆发放记录。
- 按客户账户、事件号、追踪人和时间范围批量查询满意度调研发放京豆记录。
- 自动统计总数、已查、命中、未命中、异常和跳过数量。
- 查询结果支持按状态、事件号、追踪人、客户账户、京豆创建时间筛选。
- 支持导出当前筛选下的命中结果 CSV。
- 在 CRM 业务监控明细页支持将数据导出到本地文件。

## 使用方法

1. 打开 `http://newadmin.jpos.jd.com/tool/beanList` 页面。
2. 确认浏览器已登录 CRM、京豆查询系统和所选请求来源对应系统。
3. 在工具面板的“数据范围”中选择“今天关闭量”或“昨天+今天关闭量”。
4. 点击“获取数据”，等待工具读取当前 CRM 组数据。
5. 在“人员”下拉框中选择“整组”或指定追踪人。
6. 在“请求来源”中选择 `jpos` 或 `kfuad`。
7. 设置“开始时间”和“截止时间”。
8. 点击“开始”进行批量查询。
9. 查询过程中可点击“停止”中断任务。
10. 查询完成后，可在结果表头使用筛选按钮过滤结果。
11. 有命中数据时，点击“导出CSV”导出当前筛选下的命中结果。
